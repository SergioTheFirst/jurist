# CONSTITUTION.md — Конституция проекта LegalDesk

## Статья 1. Назначение
LegalDesk — автоматизированное рабочее место юриста. Деперсонализация текста локально, юридический анализ через КонсультантПлюс API.

## Статья 2. Неприкосновенность данных
1. Персональные данные НИКОГДА не покидают локальную машину.
2. Во внешний API уходит ТОЛЬКО очищенный и одобренный юристом текст.
3. Маппинг ПДн хранится в памяти сессии или в шифрованном хранилище.
4. Нарушение этого принципа — блокирующий баг, релиз невозможен.

## Статья 3. Качество кода
1. Ни одна строка бизнес-логики без теста.
2. mypy --strict без исключений.
3. ruff без подавления правил (кроме документированных в pyproject.toml).
4. Каждый коммит проходит make lint && make test.
5. Покрытие тестами не ниже 80%.

## Статья 4. Архитектурные границы
1. Три модуля: anonymizer, legal_engine, web. Зависимости однонаправленные:
   - web → anonymizer, web → legal_engine
   - anonymizer и legal_engine НЕ зависят друг от друга
   - anonymizer НЕ обращается в сеть (только localhost Ollama)
2. legal_engine работает через абстракцию Protocol (LegalSearchProvider).
3. Web UI — Flask + Jinja2, минимальный JS, никаких внешних CDN в проде.

## Статья 5. Принцип фундамента
1. Инфраструктура (CI, линтеры, тесты) создаётся ДО бизнес-логики.
2. Новая фаза не начинается, пока предыдущая не проходит CI.
3. Рефакторинг — полноправная задача, не «техдолг на потом».

## Статья 6. Процесс разработки
1. Одна фаза — одна ветка — один PR.
2. Перед реализацией — план (/plan в Claude Code).
3. Коммиты атомарные: один коммит = одна логическая единица.
4. Сообщения коммитов: `feat:`, `fix:`, `refactor:`, `test:`, `docs:`.

## Статья 7. Безопасность
1. Никаких секретов в коде (API-ключи через переменные окружения).
2. Входные данные всегда валидируются (Pydantic).
3. SQL-инъекции невозможны (параметризованные запросы, если будет БД).
4. Зависимости минимальны, каждая обоснована.

## Статья 8. Изменение конституции
Изменения вносятся через PR с пометкой `docs: constitution` и требуют явного обоснования в описании.
