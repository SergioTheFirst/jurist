# LegalDesk — АРМ Юриста

Веб-приложение для безопасной работы с правовыми базами: автоматическое удаление персональных данных перед отправкой запросов в КонсультантПлюс.

## Возможности

- Анонимизация текста с помощью локальной LLM (Ollama) и regex-правил
- Визуальная проверка замен юристом перед отправкой
- Поиск по правовым базам через API КонсультантПлюс
- Обратная подстановка ПДн в результат

## Конфигурация

| Переменная | Описание | По умолчанию |
|---|---|---|
| `LEGALDESK_SECRET_KEY` | Секретный ключ Flask для подписи сессий | Генерируется автоматически (dev) |
| `LEGALDESK_OLLAMA_URL` | URL Ollama API | `http://localhost:11434` |
| `LEGALDESK_OLLAMA_MODEL` | Модель Ollama | `llama3.1:8b` |
| `LEGALDESK_USE_LLM` | Использовать LLM для анонимизации | `true` |

## Первый запуск

```bash
# 1. Установить зависимости
pip install -e ".[dev]"

# 2. (Опционально) Скачать модель LLM — без Ollama работает в regex-only режиме
ollama pull llama3.1:8b

# 3. Запустить сервер
make run

# 4. Открыть в браузере
# http://127.0.0.1:5000

# 5. Проверить статус системы
# http://127.0.0.1:5000/health
```

## Как работает

```
Ввод текста → Анонимизация (LLM + regex) → Проверка замен юристом → Анализ (КонсультантПлюс) → Результат
```

1. Юрист вставляет текст обращения
2. Система находит и заменяет ПДн (ФИО, ИНН, СНИЛС, телефоны и т.д.)
3. Юрист проверяет замены, может добавить или убрать
4. Очищенный текст отправляется на анализ
5. Результат отображается с найденными нормами права

## Без Ollama

Приложение полностью работает без Ollama в degraded (regex-only) режиме. При этом:
- Контекстно-зависимые данные (ФИО в падежах, адреса) могут быть пропущены
- Юрист получает предупреждение и должен подтвердить ручную проверку

Статус Ollama доступен на `/health`.
